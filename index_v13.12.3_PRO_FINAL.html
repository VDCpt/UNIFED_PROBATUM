1. Declaração de Arquitectura e Fluxo de Dados

⚠️ DECLARAÇÃO TÉCNICA OBRIGATÓRIA (para efeitos de admissibilidade pericial):

Este sistema opera como ficheiro HTML local. A bateria de 50 questões de contraditório opera exclusivamente offline — sem comunicações externas. O motor Merkle e os cálculos SHA-256 executam via WebCrypto API (SubtleCrypto) local ao browser.

Funções opcionais de análise por IA (se configuradas) transmitem dados via proxy externo (api.unifed.com → api.anthropic.com/v1/messages). O sistema NÃO é sandbox isolada nessas funções. O perito deve declarar esta arquitectura em relatório pericial ao abrigo do Art. 125.º CPP.

3. Motor Merkle SHA-256 — Selective Disclosure [INOVAÇÃO v13.12.3-PRO]

Fundamento técnico: Cada questão seleccionada gera um nó-folha SHA-256 (SHA256(q.id + q.questao)). Os nós são combinados por pares em camadas sucessivas até à raiz Merkle (Merkle Root). A raiz é exibida no rodapé como Master Hash.

Vantagem pericial: Um terceiro pode verificar criptograficamente que a questão X consta do guião sem revelar todas as outras questões (princípio de Selective Disclosure, análogo ao previsto em eIDAS 2.0 / EUDI Wallet Architecture). Esta abordagem eleva o sistema a padrão de 2030+ em admissibilidade de prova digital.

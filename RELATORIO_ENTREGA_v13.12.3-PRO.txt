📊 ESTRUTURA DE QUESTÕES (50 TOTAL)
═══════════════════════════════════════════════════════════════════════════════

EIXO A: CADEIA DE CUSTÓDIA (10 questões)
─────────────────────────────────────────
Foco: ISO/IEC 27037, integridade de dados, preservação de prova
Questões críticas:
  A01 — Conformidade ISO/IEC 27037:2012 na custódia de dados
  A03 — Hash SHA-256 de dados originais (validação independente)
  A08 — Código-fonte do algoritmo UNIFED (contra-perícia)
  A10 — Relatório técnico detalhado com screenshots

EIXO B: TRIANGULAÇÃO DAC7 vs SAF-T (10 questões)
──────────────────────────────────────────────────
Foco: Omissões declarativas, reconciliação de dados, reporte DAC7
Questões críticas:
  B01 — Reconciliação com reportagem DAC7 oficial (AT)
  B03 — Delta entre SAF-T declarado e fluxos reais
  B07 — Detecção de "silêncio algorítmico" (gaps temporais)

EIXO C: NEXUS-ZERO / APROPRIAÇÃO INDEVIDA (10 questões) ⚠️ MAIS AGRESSIVO
────────────────────────────────────────────────────────────────────────
Foco: Fluxos isentos (gorjetas, portagens), propriedade de comissão
Questões críticas:
  C01 — Validação de transferências P2P diretas (gorjetas)
  C06 — APROPIAÇÃO INDEVIDA: aplicação de comissão a fluxos isentos
  C10 — Política oficial da plataforma sobre retenção de gorjetas

EIXO D: ALGORITMO & FALIBILIDADE (10 questões)
───────────────────────────────────────────────
Foco: Margem de erro, reproducibilidade, dependências
Questões críticas:
  D01 — Taxa de erro de UNIFED (precision, recall, F1-score)
  D03 — Risco de colisão de hash SHA-256 no dataset
  D09 — Reproducibilidade 100% com mesmos inputs

EIXO E: ART. 119.º RGIT / RESPONSABILIDADE CRIMINAL (10 questões)
──────────────────────────────────────────────────────────────────
Foco: Agravamento de coimas, dolo vs negligência, prescrição
Questões críticas:
  E01 — Configuração de "omissão continuada" (>18 meses)
  E03 — Prova de dolo (intencionalidade) vs culpa (negligência)
  E07 — Possibilidade de rectificação voluntária (Art. 96.º-A)

⚠️ RESOLUÇÃO DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: Módulos não carregam
  → Verificar: <script> tags corretas em panel.html
  → Console (F12): Ver erro específico
  → Solução: Remover duplicatas de carregamento

PROBLEMA: Painel não aparece após perícia
  → Verificar: Hook de desbloqueio foi adicionado ao script.js
  → Solução: Chamar manualmente no console:
     window.UNIFED_LEGAL_TACTICS_ORCHESTRATOR.unlockPanel({...})

PROBLEMA: Acordeão não abre
  → Verificar: CSS não foi injetado
  → Solução: Hard refresh (Ctrl+Shift+R)

PROBLEMA: PDF vazio ou formatado incorretamente
  → Causa: jsPDF não carregou (requer CDN)
  → Solução: Sistema fallback HTML será ativado automaticamente
            Para jsPDF completo, adicionar ao <head>:
            <script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
